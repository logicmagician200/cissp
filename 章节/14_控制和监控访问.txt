00，简介
    访问控制是保护机密性，完整性最基本的方式


01，常见的访问控制模式（授权）   
        访问控制模型就是说明主体如何访问客体的框架。
        DAC 自主访问控制
        MAC 强制访问控制
        RBAC 基于角色的访问控制
        RuBAC 基于规则的访问控制
        ABAC 基于属性的访问控制
    

02，DAC Discretionary access control
    数据所有者决定谁能访问


03，MAC Mandatory access control
    依赖敏感标签（安全标签）

    制定敏感标签（客体有分级 security classification），同时只允许高于客体级别的用户访问（主体有许可 security clearance）。

    只有管理员可以修改客体安全级别，而不是客体的所有者。

    常用于安全级别较高的场合。

    系统通过比较主体的安全许可和客体的安全级别做出访问决策。

04，RBAC
    使用集中访问控制决定主客体的访问。
    
    特点：
        基于工作职责的权限分配
        可以和组织结构关联
        用户和组对应角色，赋予角色某些权限。
        能够遵守最小特权原则
        可以防止权限蔓延
    
    类别：
        核心 RBAC：用户、角色、权限、操作、会话应该根据策略进行定义和对应

        层次化 RBAC：角色关系定义了用户成员和权限集成
            反应组织机构和功能描述

            类型：
                有限层次：单角色继承

                普通层次：多角色继承


        受限 RBAC0：引入职责分离

            RBAC1 中的静态职责分离：会计和出纳，防止欺诈

            RBAC2 中的动态职责分离：根据激活的会话角色，动态限制另外的职责分离权限。
    
    基于角色和基于 group 
        在 group 的情况下，可以继承其他 group
        用户只能拥有当前角色的权限

    RBAC 支持的三个安全原则：
        最小特权
        职责分离
        数据抽象

05，RuBAC
    基于 if x then y
    使用特定的规则来指示主体和客体之间能发生什么，不能发生什么
    基于规则的访问控制不一定基于身份。

    许多路由器和防火墙使用规则来确定哪些类型的数据包被允许进入网络

06，ABAC
    更细颗粒度

    新型的访问控制，解决 RBAC 的不足。

    每个资源和用户都被赋予一系列的属性，基于对用户属性的评估，比如时间、职位、位置，来确定用户是否能访问某个资源。

    ABAC 模型在云计算，物联网，大数据等领域有着广泛的应用

基于风险的访问控制
    基于风险的访问控制是一种访问控制的策略，它主张根据特定的风险情况动态的对用户进行访问控制，这些访问控制的主要思想是，在某些情况下，即使用户从某种角度看是授权的，单由于存在某些风险，他们的访问仍然被拒绝。

    基于风险的访问控制通常基于多个风险因素进行评估，包括用户类型、设备类型、访问环境、网络连接性、应用程序访问信息等。基于这些风险因素，可以采取适当的控制措施，例如进行多因素认证，限制访问时间等。

    基于风险的访问控制是一种灵活和智能的访问控制方法，可以根据特定环境和用户需求进行调整。


07，常见的访问控制方法
    1，访问控制矩阵
    2，基于内容的访问控制
    3，基于上下文相关访问控制
    4，限制性用户接口

08，访问控制矩阵 access control matrix
        
        访问控制矩阵是主体和客体关系的矩阵表。

        能力表（列）：描述主体，规定了主体能够访问哪些客体

        访问控制列表 ACL（行）：描述客体，规定能够对此客体进行访问的主体有哪些。

09，基于内容的访问控制 content-dependent
        对客体的访问基于客体的内容

        例如：基于内容的过滤规则  包过滤防火墙

        例如：数据库视图

10，基于上下文相关访问控制  
        基于上下文做出访问决策

        例如：状态检测防火墙

        日期和时间控件作为上下文相关控件。

13，对口令保护的建议
        检查口令强度
        密码散列和加密
        密码生命周期
        限制登录的尝试次数

14，对智能卡的攻击

        非侵入式：
            旁路攻击：
                    差分功耗分析：查看处理过程中的功耗量
                    电磁分析：查看放射频谱
                    时序分析：计算某以特定功能所需要的时间

            故障生成：引入错误，比对正确的处理结果，以试图发现加密密钥

            软件攻击：在智能卡中输入提取用户指令的信息

        侵入式：
            微探测：使用超声波去除智能卡电路上的外部保护材料，探针直接连接智能卡的 ROM 芯片来访问和操作其中的数据。
