

01，PII 个人身份信息：任何可以识别个人的数据
        1，可用于区分或追踪个人身份的任何信息，如姓名、社会保障码、出生日期和地点、母亲婚前姓名、生物特征记录。
        2，与个相关或可链接的任何其他信息，如医疗、教育、财务和就业信息。

02，PHI 受保护的健康数据
        PHI 是与特定人相关的任何健康相关信息。
        在美国，HIPAA《健康保险便携性和责任法案》规定了 PHI 保护。

        HIPAA 提出了 PHI 的更正式的定义：
                1，由医疗保健提供者、医疗计划、公共卫生当局、雇主、人寿保险公司、学校或大学或医疗保健清算所创建或接受
                2，与任何个人过去、现在或将来的身体或心理健康或状况、向个人提供的医疗保健、或向个人提供医疗保健的过去、限制和将来的付款有关的信息。

                根据 HIPAA，雇主保存的健康信息作为员工雇佣记录的一部分不被视为 PHI。然而，管理自费健康计划的雇主必须满足某些要求，将雇佣计划和健康计划记录分开，以避免不允许的 PHI 披露。

03，GDPR 中的敏感数据类型：
        1，种族民族起源
        2，政治观点
        3，宗教或哲学信仰或工会会员身份
        4，基因数据
        5，生物特征数据
        6，有关健康的数据
        7，有关自由人的性生活或性取向的数据

04，classification 数据分级/分类
        数据分级的依据、标准：
                1，按照数据的敏感度进行分类
                2，如果数据被泄露，可能造成的损害程度

05，定期审查
        审查分类层次、数据和程序等相关内容，确保他们仍然与业务需求保持一致，数据或应用程序可能还需要重新分类。

07，责任和层级
        1，高级管理者：理解公司愿景、业务目标。

        2，下一层是职能管理者：其成员了解各自的部门如何工作，个人在公司内部扮演的角色，以及安全如何直接影响他们的部分。

        3，下一层是运营经理和员工：这一层更接近公司的实际运作，他们知道实际的技术和程序要求，系统，以及如何使用系统的信息。

        4，在这些层的员工了解安全机制集成到系统中，如何配置他们，以及他们如何影响日常生产力。

        5，每一层级，都应该输入最好的安全措施，程序和选择的控制，以确保商定的安全级别提供必要的保护。

        6，特别说明：高级管理层是组织安全的最终负责人。

09，Heartbleed 心脏滴血漏洞
        如果未能检查内存读取请求的边界，可能会将信息从一个进程暴露给一个系统上运行的其他进程。

10，Meltdown 和 Spectre
        Meltdown 和 Spectre 都是源于现代 CPU 用于预测未来指令以优化性能的方法。

        Meltdown：允许非特权进程读取私有内核内存内容的漏洞。

        Spectre：可以从其他正在运行的应用程序中大规模窃取内存内容。

11，DLP data loss prevention 数据防泄露
        DLP 是防止企业敏感信息泄露的一套系统。

        DLP 的三个关键目标：    
                1，将存储在整个企业的敏感信息进行定位和编程目标。
                2，对整个企业名敏感信息的移动进行监控和控制
                3，对终端用户系统敏感信息的移动进行监控和控制

10，DPI deep packet inspection 深度报文检测技术

        DPI solution：
                1，被动的检测网络流量
                2，识别捕获的正确数据流量
                3，组装所收集的数据
                4，在数据流中进行文件重建
                5，执行静态数据同样的分析并确认文件内容的任何部分是受到规则限制的。
        
        DPI 是 DLP 的核心技术，能够越过报文头部去阅读书籍的有效载荷。

11，隐写术
        隐写术是一种数据隐藏技术，将大量数据隐藏到图片或视频文件中。

        信息隐藏包括隐蔽通道、在 web 页面隐藏文本、隐藏可见文件、空密码。

16，消除数据残留 Remanence
        1，erasing 擦除：普通的删除磁盘数据，格式化磁盘，数据容易被恢复。
        2，clearing 清理：使用一般的软件无法恢复，但是特殊的实验室工具可以恢复。
                        Overwriting 覆写：使用程序对介质进行写操作，以覆写原有数据。通常在进行三遍。
        3，purging 清除：purging 之后的数据使用现有技术无法恢复。
                        消磁：将磁性介质暴露在强磁场环境中。
                                磁盘消磁后不能在使用
                                磁带消磁后还可以使用。
                                SSD 无法消磁

                        其他 purging 方法：覆写、块擦除、加密擦除、物理破坏。

        4，destruction 破坏、销毁：
                存储介质被破坏到常规设备无法读取和使用的地步至不可用
                从物理上销毁介质，完全销毁存储介质，最安全。

                化学焚烧，腐蚀
                物理肢解，碾碎
                形态转换，将固定转换成液体，气体等
                对于磁性介质，提高温度以超越居里温度。
        
        加密擦除：加密数据使其没有相应密钥的情况下不可读
                云存储的数据安全和数据残留问题，使用数据加密

        净化（Sanitizing）：
                1，有时泛指消除数据残留的方法
                2，实时根据上下文又指数据脱敏

        SSD 固态硬盘：
                用集成电路代替磁盘，消磁不适用 SSD。
                净化 SSD 的最佳方法是销毁，物理破坏，粉碎，加密擦除

19，data owner 数据所有者
        1，负责决定数据的分级
        2，批准数据的访问权限
        3，直接或间接决定谁可以访问特定的数据
        4，数据所有者通常具有数据的法律权限，包括知识产权和版权等。
        5，数据安全、防泄密控制、数据发布、价格、传播相关的策略
        6，信息一旦创建，必须明确所有权。
        7，为任何形式的数据泄露负责

20，资产所有者/系统所有者
        资产所有者或系统所有者是拥有处理敏感数据的资产或系统的人员。

        资产所有者和数据所有者可能是同一个人，也可以是不同的人。

21，数据处理者 Processors
        GDPR 中数据控制者决定数据的处理方式和目的

        GDPR 中数据处理者按照数据库控制者的要求对数据进行处理。

22，假名化 Pseudonymization
        假名就是别名

        假名化的个人数据依旧是个人数据。

        当向第三方发布数据集（假如研究人员聚合数据）而不向第三方发布任何隐私数据时，假名化最有用。


23，匿名化 Anonymization
        匿名化是去除所有相关数据，使得理论上无法识别数据主体或个人的过程。
        
        匿名化之后的个人数据就不在是个人数据。


24，DRM digital rights management 数字版权管理
        DRM 试图为受版权保护的作品提供版权保护，目的是防止未经授权的使用、修改和分发受保护的作品。

        DRM 保护的方法：
                1，DRM 许可证：DRM License

                2，持久在线身份验证：要求系统连接互联网才能使用，系统定期与认证服务器连接，如果连接失败，DRM 将阻止使用该产品。

                3，持续审计跟踪：跟踪版权的使用情况，与【持久在线身份验证】结合使用，可以检测滥用。

                4，自动过期：

        DRM 方法用户保护受版权保护的数据，但是不适用保护商标、专利或商业秘密。

25，令牌化 Tokenization
        令牌化使用令牌（通常是随机字符串）来替换其他数据。通常用于信用卡交易。

        令牌化类似于假名化。假名化使用假名来替换其他数据。令牌化使用令牌（token）来表示其他数据。



26，云访问安全代理 Cloud Access Security Broker CASB
        CASB 是一种逻辑上位于用于和基于云的资源直接的软件。

        CASB 可以部署在本地，也可以部署在云上。任何访问云的人都会通过 CASB 软件，它监控所有活动并执行管理员定义的安全策略。

        CASB 通常包括身份也验证和授权机制，并确保只有授权的用户才能访问云资源。

        CASB 解决方案还可以有效检测影子 IT。

31，什么是数据库维护？
        数据库被创建以后的工作都叫做数据库维护。
        包括：备份系统文件、恢复数据库系统、生产用户信息表、为信息表授权、监视系统运行状态，及时处理系统错误，保证系统数据安全，周期性更改口令
