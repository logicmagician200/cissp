03，DNS 工作原理
    每当客户端需要将DNS名称解析为IP地址时，它可以执行以下过程：
        1.检查本地缓存（包括hosts文件中的内容）。
        2.向已知DNS服务器发送DNS查询。
        3.向任何可能的本地子网DNS服务器发送广播查询。（此步骤不受广泛支持。）
        
        如果客户端没有从这些步骤中获得DNS到IP的解析，则解析失败，无法发送通信。有许多方法可以攻击或利用DNS，其中大多数方法用于返回错误结果。

05，DNSSEC
    DNSSEC 加强了 DNS 的身份验证机制。

    开发 DNSSEC 技术的主要目的是通过对数据进行“数字签名”来确保完整性。

    DNSSEC 是对 DNS 的安全扩展。
    DNSSEC 的主要功能是在 DNS 操作期间提供设备之间的相互证书认证。

    DNSSEC 仅适用于 DNS 服务器之间，而不是适用于对 DNS 服务器执行查询操作的客户端。

09，SDN software define network
    将网络设备的控制器分离出来，由集中的控制器进行管理。无需依赖底层网络设备，屏蔽了来着底层不同厂商网络设备的差异。

    网络虚拟化：允许数据传输路径、通信决策树、流量控制等在 SDN 的控制层中虚拟化，而不是基于每个设备的硬件。

    可以从集中位置编程，灵活，基于开发标准

    独立于供应商：允许组织根据需要混合和匹配硬件，而不是被某个供应商锁定。

10，SDP software defined perimeter 软件定义边界
    SDP 的网络隐身技术可以很好的实现 never trust 原则。

    在传统的 TCP/IP 网络中，是先连接后验证，这样不可避免的将网络资源暴露在公网下。

    SDP 采用先验证后连接，在没有验证身份之前，服务端对客户端是不可见的，从 by default trust 变成了 never trust。

    传统的网络是 verify once，SDP 实施动态可信授权验证，实现 always verify。

    SDP 是零信任的快速高效的零信任安全实践技术架构，因此被行业广泛应用。

    SDP 的 ABCD 原则：
        A：Assume nothing 不假设任何事情
        B：Believe nobody 不相信任何人
        C：Check everything 检查所有内容
        D：Defect threats 阻止威胁

11，微隔离/微分段 Microsegmentatioin
    NIST 白皮书总结了几种实现零信任的方式，SDP技术实现南北向安全，微隔离实现东西向安全。微隔离是零信任架构的重要组成部分。

    网络隔离概念的演变是微隔离。微隔离将内部网络划分为许多子区域，可能小到单个设备，例如高价值服务器甚至客户端或端点设备。

    每个区域都通过内部分段防火墙（ISFW）、子网、VLAN或其他虚拟网络解决方案与其他区域分开。区域之间的所有通信都经过过滤，可能需要进行身份验证，通常需要会话加密，并且可能受到黑白名单的控制。

    微隔离是实现零信任的关键。

14，防火墙：通过风险评估，确定防火墙部署的位置、数量、和具体策略
    包过滤防火墙：第一代防火墙
        工作站网络层
        难以防范对上层协议的攻击
    
    应用代理防火墙：第二代防火墙
        工作在应用层
        监视应用层协议，并且以自身的名义转发，通信双方没有直接路由。

    电路级网关防火墙：
        工作在会话层
        【包过滤】和【应用代理】的混合体
    
    状态检测防火墙：第三代防火墙
        工作在网络层、传输层
        维护一个状态表，以跟踪记录每一个通信通道。
        跟踪 UDP、TCP 数据包

        所谓的“状态检测”是指 检测TCP传输的连接状态或是UDP传输的虚连接状态（UDP是无连接的，所以状态检测防火墙会为UDP建立一个虚连接）。

        状态检测防火墙会维护一个连接表，为每个TCP连接或UDP虚连接建立一条会话记录，这条会话记录包含此连接的源地址、源端口、目标地址、目标端口、连接时间等信息，对于TCP连接，它还应该会包含序列号和标志位等信息。

        当后续数据包到达时，状态检测引擎就会把它的相关信息与状态表中的会话记录进行比较，如果信息匹配，就直接允许数据包通过，如果信息不匹配，数据包就会被丢弃或连接被拒绝，并且每个会话还有一个超时值，过了这个时间，相应会话记录就会被从状态表中删除掉。这是状态检测防火墙的主要工作原理。

    动态包过滤防火墙：第四代防火墙
        ACL 是动态的，连接结束后销毁。

        有人认为【状态检测防火墙】和【动态包过滤防火墙】都是第三代防火墙

    内核代理防火墙：第五代防火墙
        评估数据包，防火墙会建立动态的、定制的 TCP/IP 协议栈。    

    下一代防火墙 NGFW
        1，改进现有防火墙静态策略的局限性
        2，引入外部动态数据资源
        3，开发一个程序保证在恰当的时间采用恰当的规则到达恰当的位置。
